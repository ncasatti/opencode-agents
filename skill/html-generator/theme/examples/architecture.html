CLINGY.

SYSTEM SPECIFICATION

The Clingy framework is a thin discovery layer over a directory tree. There is no daemon, no socket, no cache. Each invocation reparses the surrounding tree and produces a menu graph. This document specifies the lattice that turns a folder into a command surface.

01. Core Architecture Design

The framework operates as a stateless command resolver. On invocation, a single shell function walks the project tree, indexes executables tagged by sigil, and presents them through fzf as a navigable menu.

Discovery is bounded by directory traversal cost; no in-memory index is preserved across invocations. This deliberately trades cold-start speed for absolute consistency with the on-disk state of the project.
